Pre-seed an admin token on the CI Infrahub stack (`INFRAHUB_INITIAL_ADMIN_TOKEN` in `development/docker-compose-ci.yml`) and pass the matching `INFRAHUB_TOKEN` env to the `schema-validation` and `integration-infrahub` jobs in `.github/workflows/quality.yml`. `load_schemas.py` and `populate_sot.py` already read the env via their existing `--token` flag, so this unblocks the chronic HTTP 401 in both jobs without script changes.
