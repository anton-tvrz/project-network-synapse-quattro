Per-device workflow concurrency control: all device-mutating workflows (change, drift remediation, emergency change, override) now start through `synapse_workers.triggers.start_device_workflow`, which derives one shared Temporal workflow ID per device (`device-ops-<hostname>`) with an explicit reject-while-running conflict policy. Two operations can no longer mutate the same device concurrently — a drift remediation can't fire mid-change and corrupt the backup baseline. The self-service portal returns HTTP 409 when a device is busy, and the task-queue name is single-sourced in the same module.
