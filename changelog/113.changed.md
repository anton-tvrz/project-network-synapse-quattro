Hardened gNMI activity I/O: the synchronous pygnmi client used by `fetch_running_config`, `backup_running_config`, `deploy_config`, and `rollback_config` is now offloaded onto a worker thread via `asyncio.to_thread`, so a slow device no longer blocks the Temporal worker's event loop. The duplicated GET logic in `drift_activities` and `device_backup_activities` is consolidated behind a shared `_gnmi_io.fetch_config_via_gnmi` helper, and transport errors (gNMIException, grpc.RpcError, ConnectionError, OSError, TimeoutError) are caught explicitly and rewrapped as RuntimeError for consistent activity failures.
