Bind the Infrahub server to `0.0.0.0` in both `development/docker-compose-deps.yml` and `development/docker-compose-ci.yml` so the listener is reachable from the host (the CLI default of `127.0.0.1` made `uv run invoke dev.deps` and the CI `Wait for Infrahub health` step unable to talk to the container). Also pin the local compose stack to `name: synapse-quattro` so it does not collide with another project that happens to use a `development/` directory.
