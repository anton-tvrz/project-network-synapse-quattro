Device backup/rollback now works against real SR Linux: backups GET only the writable config datastore (`datatype="config"`) instead of the full tree whose operational leaves SR Linux rejects on SET, all GET updates are merged into the backup instead of keeping only the first, and rollbacks push with gNMI `replace` so the device returns to exactly the backed-up config rather than merging over the failed change. Verified end-to-end against a live containerlab node (`tests/e2e/test_rollback_restores_device.py`). Drift remediation's config fetch shares the same fix.
